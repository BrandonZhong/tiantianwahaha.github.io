Arbitrary Style Transfer in Real-time with Adaptive Instance Normalization

Arbitrary Style Transfer in Real-time with Adaptive Instance Normalization

论文地址

作者源码实现Torch

较早的一篇效果不错的任意风格任意内容的风格转换模型。

主要贡献就是提出了AdaIN（Adaptive Instance Normalization）自适应实例归一化，nvidia的stylegan里用了这个结构

没仔细读论文，简单看了两篇博客。

模型结构

模型包括三部分Encoder-AdaIN-Decoder。

Encoder 部分是采用预训练好的VGG网络，只使用到了Relu4_1部分，将风格和内容图的图像都从图像空间转到特征空间。

AdaIN层是对内容图进行归一化，这里是通过对齐Relu4_1层输出的内容图对应的每通道的feature map的均值和方差来匹配风格图每通道feature map的均值和方差。就是把内容图每个通道的所有元素归一化到风格图对应通道上。

Decoder部分是一个将feature 空间转成图像空间的网络，这部分网络一般是采用和encoder对称的网络结构，整个网络中需要训练的就是这部分网络的权重参数信息，初始可以随机一些初始化参数，通过梯度下降可以不断进行更新参数以使整个损失函数下降、网络逐渐收敛。池化层一般是替换成采用最近邻上采样的方式来防止棋盘效应，在encoder 和 decoder部分的padding一般都是采用反射填充避免边界artifacts。decoder中没有使用归一化层，因为IN/BN这些实例归一化和批归一化都是针对单个风格的

损失函数

就是内容损失+风格损失

风格损失是转换后图像在vgg网络中Relu4_1的特征和adaIN输出feature maps的欧式距离

内容损失，没用使用Gram矩阵，简单的采用了relu1_1,relu2_1,relu3_1,relu4_1四层的feature maps，求对应的特征距离损失。

结果图像质量超过原来的文章或者差不多，但是速度快了1-2个量级
